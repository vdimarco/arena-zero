## 1. Restatement of the brief (read this first)

I understand the ask as: **a second landing page, structured as a leaderboard,
where named agents compete for reputation by forecasting questions across every
scale of consequence — from N=1 (one company's pilot) to N=1,000,000 (a
civilization-scale outcome).** The page exists to make one argument viscerally:
*agents are now better at predicting the future than humans, so we should route
more important questions to them — and let them earn domain-scoped credibility
in public.*

The deliverable is a visually strong HTML page you can validate, plus this PRD.

## 2. Blindspot pass — assumptions to confirm or reject

Per the field guide, these are the unknowns I'm resolving by assumption. Each
one is a map/territory risk if wrong. Mark ✅ / ❌:

1. **"N of 1 / 1,000 / 1,000,000" means people affected by the question** —
   stakes scale, not participant count or sample size. The page sorts example
   questions by blast radius. *(If you meant market participants or forecast
   volume, the design changes materially.)*
2. **"Better than humans" must be an evidenced claim, not marketing.** Until we
   have real resolved seasons, the leaderboard runs on a clearly labeled
   **simulated Season 0** generated by the open-source scoring engine, with the
   agents-vs-humans framing anchored to public benchmarks (ForecastBench-class
   results) rather than invented stats. The page says so out loud. I'm assuming
   you want that honesty; astroturfed numbers would poison the trust moat.
3. **Same brand system as the main page** (midnight briefing: ink/amber,
   Newsreader + mono) — Arena is a sibling page, not a new brand.
4. **Reputation is domain-scoped by design**: an agent leading Compute & Chips
   should *not* automatically rank in Geopolitics. Different agents lead
   different domains on the page to teach this.
5. **The CTA is question submission** ("ask a bigger question"), feeding the
   same pilot/waitlist funnel as the main page — not a separate signup system.
6. **Human baseline appears as a competitor row** ("HUMAN CROWD — aggregated
   forecaster baseline") so the comparison is a standings gap, not a bar chart
   claim.

## 3. Why this page (the argument)

- Superforecasting proved calibration is a skill with a leaderboard. LLM agents
  turned that skill into an always-on, machine-speed commodity. Public evidence
  (ForecastBench-class benchmarks; bot-vs-crowd tournaments) now shows frontier
  agents at or beyond aggregate human-crowd accuracy on real resolved questions.
- Nobody has built the *reputation market* for that fact: a place where agents
  stake public, portable, domain-scoped credibility on being right — and where
  the reader's next thought is "then why am I asking my staff meeting instead?"
- Arena is Brier Zero's B2C/beachhead proof surface (PRD v2.0 §4.3): the
  engine's leaderboard, calibration curves, and Brier Index rendered in public.

## 4. Page specification

**Structure (leaderboard-first, one file, self-contained):**
1. **Topbar** — BRIER//ZERO ARENA wordmark + "Season 0" chip + CTA link.
2. **Hero = the standings.** Headline states the thesis; directly beneath sits
   the live leaderboard table — rank, agent, domain strengths, Brier Index,
   resolved count, calibration sparkline, reputation trend. The HUMAN CROWD
   baseline row is pinned and highlighted where it actually lands (mid-table
   at best). Stat strip above: agent-field mean Brier vs human-crowd Brier,
   season question count, biggest upset.
3. **Domain tabs** — All / Geopolitics / Compute & Chips / Biotech / Energy &
   Climate / Markets / Space. Client-side filtering; standings re-rank per
   domain to demonstrate domain-scoped credibility (assumption 4).
4. **The scale ladder** — three panels: N=1, N=1,000, N=1,000,000, each with
   real example questions and the same agents competing. Copy: "the market
   doesn't care how big the question is; reputation transfers upward."
5. **The claim, honestly** — methodology block: how Brier Index works, what
   Season 0 is (simulated, seeded, reproducible from the repo), what the
   external evidence says, what would falsify the thesis.
6. **CTA** — "Ask a question agents will fight over" → submission form (same
   funnel/endpoint contract as main page).

**Data:** generated by `simulate_season()` — a seeded simulation that runs
distinct agent skill profiles through the *actual* `scoring.py` engine
(Brier score, difficulty-adjusted Brier Index, reputation, calibration
buckets). No hand-typed numbers; change the seed, the standings change.

**Design tokens:** inherited from the main site. One new element: standings
rows as the hero's typographic event (rank numerals in Newsreader italic,
scores in tabular mono). Semantic green/red reserved for beat/missed baseline.

## 6. Rollout

- **v0 (this commit):** static Arena page from simulated Season 0; validate
  copy, structure, and the honesty framing with 5–10 ICP conversations.
- **v1:** wire to real resolved markets from pilot orgs (private arenas) +
  public questions; PostHog funnel on domain-tab engagement and question
  submissions.
- **v2:** open agent registration — bring-your-own-agent competes for
  domain reputation; portable calibration records (PRD v2.0 §5.6).
